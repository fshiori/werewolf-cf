## 2️⃣ 潛在 Bug (6 個)

1. **Durable Object 狀態持久化可能不完整**
   - 儲存/載入時機不確定
   - 需要測試重啟後狀態保留

2. **WebSocket 重新連線處理可能缺失**
   - 斷線後需要自動重連
   - 需要同步遺失的訊息

3. **前端 innerHTML 可能存在 XSS 風險**
   - 使用者輸入未完全轉義
   - 需要使用 textContent 或徹底轉義

4. **缺少請求速率限制**
   - API 可能被 DDoS
   - WebSocket 連線可能被濫用

5. **Tripcode 生成使用簡單 base64**
   - 不夠安全，應使用 crypt() 或 bcrypt

6. **Session 過期清理依賴手動觸發**
   - 需要自動清理機制
   - 或設定 TTL 自動過期

## 4️⃣ 安全性強化 (7 個)

1. **實作 CSRF 保護**
   - Token 驗證
   - SameSite Cookie

2. **加入請求速率限制**
   - API rate limiting
   - WebSocket connection limit

3. **強化 Tripcode 演算法**
   - 使用 crypt() 或 bcrypt
   - 加鹽 (salt)

4. **XSS 防護**
   - 轉義所有使用者輸入
   - Content-Security-Policy

5. **SQL 注入防護**
   - 使用參數化查詢
   - 輸入驗證

6. **Session 劫持防護**
   - 加入 IP 檢查
   - User-Agent 驗證

7. **實作 Content Security Policy (CSP)**
   - 限制腳本來源
   - 防止 XSS
